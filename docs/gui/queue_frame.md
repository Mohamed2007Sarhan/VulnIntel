# `gui/queue_frame.py`

## الدور

**طابور معالجة يدوي** لـ CVE: إضافة CVE إلى قائمة، خيارات (Fetch، Exploits، Sigma، YARA، IOC)، وتشغيل المعالجة في خيط خلفي حتى لا تجمد الواجهة.

## التبعيات

`DatabaseManager`, `CVEAnalyzer`, قاموس `sources`, واختياريًا `AIAnalyzer`.

## الشكل

- رأس مع شارة حالة الطابور (IDLE / BUSY) وعدّاد عناصر.  
- منطقة إدخال CVE + خانات اختيار الإجراءات.  
- قائمة العناصر المنتظرة/المكتملة، وأزرار تشغيل/مسح حسب التطبيق.

## مسار البيانات

عند التشغيل، يستدعي دوال المصادر والمحلل مشابهة للـ pipeline لكن لـ CVE يختاره المستخدم.
